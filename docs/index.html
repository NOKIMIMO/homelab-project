Fonctionnalités

Tout ce dont tu as besoin,
rien de superflu

Le backend gère l'infrastructure. Toi tu déclares ce que tu veux faire.

📦

Système de modules

Chaque module est un dossier avec un manifest.json. Le backend le découvre, crée sa base de données et l'expose automatiquement.

🗄️

Base de données auto-générée

Déclare tes colonnes dans un fichier XML — le backend génère la table SQL, les contraintes et les relations sans aucun ORM à configurer.

🖼️

UI déclarative

Ton interface est un fichier JSON. Composants prêts à l'emploi : listes, formulaires, upload, galerie, modale, carousel — aucun JSX à écrire.

📊

Télémétrie système

CPU, RAM, stockage et uptime en temps réel via OSHI. Rafraîchissement automatique toutes les 10 secondes côté backend.

🔐

Auth JWT intégrée

Inscription soumise à validation admin, login par token JWT. Panel d'administration : logs en direct, gestion des accès, paramètres globaux.

⚙️

Paramètres par module

Chaque module peut exposer des paramètres configurables depuis l'UI (clés API, URLs, secrets masqués) sans redémarrage ni recompilation.

Fonctionnement

Zéro code
pour un nouveau module

Du dossier vide à un module fonctionnel en quelques fichiers texte.

1

Créer le dossier

Un dossier dans HOMELAB_MODULES_SCAN_PATH. Le nom devient l'ID du module.

2

Déclarer le manifest

Un manifest.json qui liste les fonctions, leurs types d'actions et les fichiers XML associés.

3

Définir le schéma XML

Le backend génère automatiquement les tables SQL à partir des colonnes déclarées. Contraintes, types, relations inclus.

4

Écrire l'UI en JSON

Liste les composants visuels, lie les bindings aux fonctions, définit le state et les actions chaînées.

5

Redémarrer le backend

Le module apparaît dans le dashboard. Démarrage, arrêt, paramètres — tout depuis l'UI.

// manifest.json { "id": "tasks", "name": "Tâches", "actions": [{ "visual": "tasks.xml", "functions": [ { "name": "createTask", "logic": [{ "type": "CREATE" }], "actUponObject": "tasks.xml" }, { "name": "listTasks", "logic": [{ "type": "LIST" }], "actUponObject": "tasks.xml" } ] }], "page": "tasks_ui.json", "dataObjects": ["tasks.xml"] }
<!-- tasks.xml --> <tasks> <autoGenerated> <id/> <createdAt/> <updatedAt/> </autoGenerated> <col> <naming val="title"/> <typing val="string"/> <nullable val="false"/> </col> </tasks>

Référence

Types d'actions disponibles

Chaque fonction de module est liée à un type d'action. Aucun code Kotlin à écrire pour les types standards.

LIST Toutes les lignes de la table
READ Une ligne filtrée par ID
CREATE Insérer une nouvelle ligne
DELETE Supprimer par filtre EQUAL
UPLOAD_FILE Upload binaire + entrée DB
GET_FILE Retourne un fichier binaire
FETCH_EXTERNAL API HTTP externe → upsert DB
